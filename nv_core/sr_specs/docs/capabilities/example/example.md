# Example Capability

## Description
This Capability is a proof-of-concept example to demonstrate the functionality of the Configurable CIP.

## Requirements
This Capabilty covers the following validation requirements:
| Code | Name | Description |
| ---- | ---- | ----------- |
| EX.01 | has-mesh | The asset contains one or more meshes. |
| EX.02 | basic-materials | There is one basic material that is bound to all meshes. |
| EX.03 | test-prim-exists | There is a child of the default prim called TestBlob. |
